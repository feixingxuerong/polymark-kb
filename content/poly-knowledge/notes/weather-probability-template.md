# 天气概率推导与市场对比模板

> 将气象预报信息转化为概率区间，并与市场隐含概率对比，发现价值偏离。

---

## 1. 概率推导核心框架

### 1.1 从预报到概率

气象预报通常给出：
- **确定性预报**：明天 35°C (点估计)
- **概率预报**：降水概率 30% (概率)
- **集合预报**：51 个成员的温度分布 (分布)

**转换方法**：

```
确定性预报 → 概率区间：
- 官方预报 "超过 35°C" → 查看历史准确率 → 修正概率
- 集合预报均值 + 标准差 → 正态分布假设 → P(X > threshold)

概率预报 → 市场概率：
- 官方降水概率 30% → 对比历史准确率 → 调整
- 检验市场是否高估/低估
```

### 1.2 概率校准 (Probability Calibration)

气象概率需要校准才能用于市场对比：

| 官方概率 | 历史命中率 | 校准系数 |
|----------|------------|----------|
| 10% | 8% | 0.8 |
| 30% | 28% | 0.93 |
| 50% | 48% | 0.96 |
| 70% | 72% | 1.03 |
| 90% | 92% | 1.02 |

**校准公式**：
```
校准后概率 = 历史命中率 / 官方预报概率 (或反之)
```

## 2. 市场隐含概率获取

### 2.1 从 Polymarket 获取

```python
import requests

def get_implied_probability(token_id):
    """获取市场隐含概率 (中间价)"""
    url = f"https://clob.polymarket.com/midpoints?token_id={token_id}"
    resp = requests.get(url).json()
    return resp.get("midpoint", 0.5)  # 默认 50%

def get_last_trade(token_id):
    """获取最新成交价"""
    url = f"https://clob.polymarket.com/last-trade-price?token_id={token_id}"
    resp = requests.get(url).json()
    return resp.get("price", 0.5)
```

### 2.2 隐含概率 vs 实际概率

- **隐含概率**：市场交易形成的概率 (包含风险溢价)
- **实际概率**：基于信息的真实概率
- **偏差**：市场可能系统性高估/低估某些类型事件

## 3. 对比分析模板

### 3.1 基础对比表

| 要素 | 内容 |
|------|------|
| **市场问题** | [粘贴完整市场描述] |
| **结算条件** | [时间/地点/阈值/数据源] |
| **市场隐含概率** | Yes: XX% / No: YY% |
| **官方预报概率** | XX% (来源: [机构]) |
| **校准后概率** | XX% (考虑历史准确率) |
| **偏差** | 市场 - 校准 = ±XX% |
| **偏差方向** | 高估 / 低估 / 一致 |

### 3.2 偏差分析框架

| 偏差类型 | 可能原因 | 应对策略 |
|----------|----------|----------|
| 市场概率 > 校准概率 | 风险溢价、博彩偏好、流动性偏差 | 考虑做空 (如果不交易则记录) |
| 市场概率 < 校准概率 | 信息劣势、流动性不足、恐慌/乐观 | 考虑做多 (如果不交易则记录) |
| 偏差稳定 | 系统性偏差存在 | 持续记录，验证偏差一致性 |
| 偏差随机 | 噪声、短期波动 | 忽略，追踪后续 |

## 5. 复盘框架

### 5.1 结算后复盘

每次市场结算后，必须记录：

```markdown
## 复盘: [市场名称]
- **结算结果**: Yes / No (实际发生了什么)
- **市场概率**: XX% (结算前最后价格)
- **我的预测**: YY%
- **偏差**: |XX% - YY%| = ZZ%
- **分析**:
  - 预测准确吗？
  - 偏差来源是什么？(数据/模型/市场情绪)
  - 学到什么？
- **改进**: 下次如何避免同类错误
```

### 5.2 典型错误分类

| 错误类型 | 描述 | 示例 |
|----------|------|------|
| **数据错误** | 搞错了站点/时间/阈值 | 把北京温度当成上海 |
| **模型错误** | 误判气象趋势 | 低估厄尔尼诺影响 |
| **市场错误** | 低估市场情绪 | 高温市场博彩偏好 |
| **校准错误** | 历史准确率估计偏差 | 官方30%实际40% |

## 6. 实际案例推导

### 案例：北京 7 月最高温 > 40°C

**Step 1: 获取市场信息**
- 市场: "Will Beijing exceed 40°C in July 2026?"
- 隐含概率: Yes 35%

**Step 2: 获取气象预报**
- ECMWF 7 月集合预报: 均值 37.2°C，标准差 2.1°C
- 历史同期极值: 41.0°C (2014年)

**Step 3: 概率推导**
```python
import scipy.stats as stats

# 假设温度服从正态分布
mean = 37.2
std = 2.1
threshold = 40.0
prob = 1 - stats.norm.cdf(threshold, mean, std)
# P(temp > 40) ≈ 9%
```

**Step 4: 校准**
- 历史 ECMWF 7 月预报准确率: 约 0.85
- 校准后概率: 9% / 0.85 ≈ 10.6%

**Step 5: 对比**
- 市场概率: 35%
- 校准概率: ~11%
- **偏差: +24%** → 市场严重高估

**决策**: 不交易 (仅记录分析)

## 7. 持续改进机制

### 7.1 跟踪指标

- **预测准确率**: 我的概率 vs 实际结果
- **偏差方向**: 市场是否系统性地高估/低估
- **信息源质量**: 哪些数据源更可靠

### 7.2 迭代优化

1. **每周**：回顾本周所有预测
2. **每月**：分析偏差模式，调整概率模型
3. **每季**：更新校准系数 (历史准确率会变化)

---

## 8. 注意事项

1. **不交易原则**：本模板仅用于研究和记录，不涉及实际下单
2. **风险提示**：气象预测本身有不确定性，概率仅作参考
3. **持续学习**：天气市场是复杂系统，需不断迭代认知
4. **数据源验证**：任何概率都必须基于可验证的数据源
