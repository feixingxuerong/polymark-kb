## 概述

- **关键口径字段**：站点、时间窗、指标定义
- **推荐监控源**：对应 Issue #22 的数据源
- **常见坑/风险清单**：边界条件、数据缺失、站点变更等风险点

**前置文档**：

- `aviation-weather-signals.md` - 航空数据信号定义

## 模板 1：最高温度 (Daily High Temperature)

### 1.1 关键口径字段

| 字段 | 示例 | 说明 |
|------|------|------|
| **站点** | 北京南郊观测站 (54511) | 需 WMO/GHCN 站点代码 |
| **时间窗** | 2026-07-15 02:00 - 次日 02:00 CST | 明确本地时区 |
| **阈值** | ≥ 40.0°C | 包含边界值处理 |
| **数据源** | CMA 国家级气象站 | NOAA/NCEI GHCN-Daily |
| **单位** | °C（需注明 °F 换算） | 避免歧义 |

### 1.2 推荐监控源

| 优先级 | 数据源 | 用途 |
|--------|--------|------|
| P0 | NOAA/NCEI GHCN-Daily | 官方结算依据 |
| P1 | CMA 国家级站数据 | 中国站点首选 |
| P1 | Open-Meteo API | 快速预览/验证 |
| P2 | Meteostat | 历史对比 |

**对应 Issue #22 映射**：
- 温度类 → Open-Meteo, NOAA/NCEI

### 1.3 常见坑/风险清单

**边界条件风险**：
- [ ] 40.0°C 是否算"超过40°C"？→ 需明确 > / ≥
- [ ] 数据保留几位小数？→ CMA 通常 0.1°C
- [ ] 四舍五入规则？→ 官方数据通常向下取整

**站点风险**：
- [ ] 站点迁移/更名 → 历史数据连续性
- [ ] 城市站 vs 机场站 → "北京"指哪个站？
- [ ] 站点关闭/无数据 → 备用站点方案

**时间窗风险**：
- [ ] 夏令时切换 → 跨日时间窗需特殊处理
- [ ] 时区混淆 → UTC vs 本地时
- [ ] "明天"歧义 → 需明确是哪个"明天"

**数据风险**：
- [ ] 数据延迟发布 → 结算时数据未出
- [ ] 数据修正 → 后续修订如何处理
- [ ] 多源冲突 → 两站数据矛盾听谁的

### 2.1 关键口径字段

| 字段 | 示例 | 说明 |
|------|------|------|
| **站点** | 哈尔滨太平机场 (ZYHB) | 机场数据更权威 |
| **时间窗** | 2026-01-15 08:00 - 20:00 CST | 日最低通常出现在清晨 |
| **阈值** | ≤ -20.0°C | 严寒预警 |
| **数据源** | CMA 机场气象站 | 需 METAR/SPECI 报文 |
| **单位** | °C（需注明 °F 换算） | 避免歧义 |

### 2.2 推荐监控源

| 优先级 | 数据源 | 用途 |
|--------|--------|------|
| P0 | NOAA/NCEI GHCN-Daily | 官方结算依据 |
| P1 | NWS METAR/SPECI | 美国机场实时数据 |
| P1 | Open-Meteo API | 快速预览 |
| P2 | 各国气象局机场站 | 本地数据 |

### 2.3 常见坑/风险清单

**边界条件风险**：
- [ ] -20.0°C 是否算"低于-20°C"？→ 明确 < / ≤
- [ ] "霜冻"定义 → 0°C vs 32°F 临界值
- [ ] 观测间隔 → 是否包含自动站数据

**站点风险**：
- [ ] 机场站 vs 城市站 → 城市热岛效应差异可达 3-5°C
- [ ] 海拔差异 → 山地/盆地站数据异常
- [ ] 站点迁移 → 需验证数据连续性

**时间窗风险**：
- [ ] "夜间"定义 → 日落至日出？
- [ ] 跨日低温 → 0:00-6:00 还是完整日历日？

### 3.1 关键口径字段

| 字段 | 示例 | 说明 |
|------|------|------|
| **站点** | 上海徐家汇 (58362) | 城市核心站 |
| **时间窗** | 2026-08-15 全日 (00:00-24:00 CST) | 明确是日历日 |
| **区间** | 30°C - 35°C | 需明确开/闭区间 |
| **指标** | 日最高与日最低之差 | 温度日较差 |
| **数据源** | CMA 日数据 | 或 NOAA LCD |

### 3.2 推荐监控源

| 优先级 | 数据源 | 用途 |
|--------|--------|------|
| P0 | NOAA/NCEI GHCN-Daily | 官方结算依据 |
| P1 | CMA 日数据 | 中国站点首选 |
| P1 | Open-Meteo | 历史/预报数据 |
| P2 | Meteostat | Python SDK 集成 |

### 3.3 常见坑/风险清单

**区间定义风险**：
- [ ] "30-35°C" 是开区间 (30,35) 还是闭区间 [30,35]？
- [ ] 包含 30.0°C 和 35.0°C 吗？→ 需明确
- [ ] 区间 vs 区间内任意一点 → 理解歧义

**指标风险**：
- [ ] "最高温度35°C" vs "温度达到35°C" → 峰值 vs 持续
- [ ] 瞬时值 vs 时段平均 → 混淆导致误判

## 模板 4：降水 - 是否 (Precipitation: Yes/No)

### 4.1 关键口径字段

| 字段 | 示例 | 说明 |
|------|------|------|
| **站点** | 纽约肯尼迪机场 (KJFK) | 机场数据权威 |
| **时间窗** | 2026-03-15 12:00-18:00 EST | 6小时降水 |
| **阈值** | ≥ 0.01 inch (0.25mm) | 微雨定义 |
| **指标** | 有降水记录 (Trace) | trace 算"有"？ |
| **数据源** | NOAA/LCD | 或 NWS METAR |

### 4.2 推荐监控源

| 优先级 | 数据源 | 用途 |
|--------|--------|------|
| P0 | NOAA/LCD | 美国官方日数据 |
| P0 | NWS METAR | 实时观测 |
| P1 | CMA 日数据 | 中国站点 |
| P1 | Open-Meteo | 全球预报/历史 |

### 4.3 常见坑/风险清单

**微量降水风险**：
- [ ] Trace (痕量降水) 算不算"有降水"？→ 各国定义不同
- [ ] 0.00mm vs 0.01mm → 仪器精度
- [ ] 雾、露水 → 算不算"降水"？

**时间窗风险**：
- [ ] "今天有雨" → 00:00-24:00 还是 06:00-18:00？
- [ ] 降水何时发生？→ 夜间降水可能漏报
- [ ] 跨时区 → EST vs EDT

**数据风险**：
- [ ] 自动站 vs 人工站 → 微量降水捕获差异
- [ ] 降水类型 → 雨/雪/冰粒定义不同
- [ ] 仪器故障 → 数据丢失

**数据风险**：

- [ ] 阵雨 vs 持续雨 → 时段选择关键

## 模板 6：风速 - 阵风/平均 (Wind: Gust/Average)

### 6.1 关键口径字段

| 字段 | 示例 | 说明 |
|------|------|------|
| **站点** | 上海浦东机场 (ZSPD) | 机场风数据权威 |
| **时间窗** | 2026-09-16 14:00-18:00 CST | 台风影响时段 |
| **阈值** | ≥ 17.2 m/s (8级风) | 或 33 kt |
| **指标** | 阵风 (Gust) 或平均风速 (Mean) | 需明确 |
| **数据源** | NWS METAR / CMA METAR | 实时报文 |

### 6.2 推荐监控源

| 优先级 | 数据源 | 用途 |
|--------|--------|------|
| P0 | NWS METAR | 美国/全球机场 |
| P0 | CMA METAR | 中国机场 |
| P1 | Open-Meteo | 网格数据 |
| P2 | NOAA/NCEI | 历史小时数据 |

### 6.3 常见坑/风险清单

**风速类型风险**：
- [ ] 阵风 vs 平均 → 差值可达 50%+
- [ ] 2分钟平均 vs 10分钟平均 → WMO 标准
- [ ] 最大阵风 vs 持续风速 → 理解歧义

**单位风险**：
- [ ] m/s vs km/h vs mph vs knots → 换算复杂
- [ ] 蒲福风级 → 对应数值需明确
- [ ] 1 m/s = 3.6 km/h = 2.237 mph = 1.944 kt

**站点风险**：
- [ ] 机场 vs 城市 → 建筑遮挡差异
- [ ] 风向影响 → 跑道方向 vs 自然风
- [ ] 地形狭管效应 → 山谷/海峡风异常

| **阈值** | 能见度 < 200m | 低能见度标准 |
| **指标** | 最小能见度 / 平均能见度 | 需明确 |
| **数据源** | NWS METAR | 或 CAA |

### 7.2 推荐监控源

| 优先级 | 数据源 | 用途 |
|--------|--------|------|
| P0 | Flightradar24 Disruption | 实时 disruption |
| P0 | NWS METAR | 实时观测 |
| P1 | FAA OPSNET | 美国延误数据 |

### 7.3 常见坑/风险清单

**能见度风险**：
- [ ] 200m vs 1/4 mile → 单位混用
- [ ] 水平能见度 vs 垂直能见度 → 雾 vs 云
- [ ] RVR (跑道视程) vs 地面能见度 → 航空专用

**云底高度风险**：
- [ ] 米 vs 英尺 → 换算 1m = 3.281ft
- [ ] 云底 vs 云顶 → 落地标准看云底
- [ ] 多层云 → 以最低层为准

**航空影响风险**：
- [ ] ILS 类别 → CAT I/II/III 标准不同
- [ ] 进近失败 → 能见度达标但云底不足
- [ ] 复飞率 → 天气标准未达标

## 模板 8：航班延误/取消 (Flight Delay/Cancellation)

### 8.1 关键口径字段

| 字段 | 示例 | 说明 |
|------|------|------|
| **机场** | 东京都市机场 (RJTT) | 需 ICAO 代码 |
| **时间窗** | 2026-07-20 全日 | 台风影响日 |
| **阈值** | 取消率 > 20% 或 延误 P90 > 120min | 需明确指标 |
| **指标** | 取消率 / 平均延误 / Disruption Index | 需明确 |
| **数据源** | Flightradar24 / Aviationstack | 需明确 |

### 8.2 推荐监控源

| 优先级 | 数据源 | 用途 |
|--------|--------|------|
| P0 | Flightradar24 Disruption | 实时 disruption (0-5) |
| P0 | FAA OPSNET | 美国官方延误统计 |
| P1 | Aviationstack API | 程序化航班状态 |
| P1 | FlightAware | 实时追踪 |
| P2 | BTS | 历史统计 |

### 8.3 常见坑/风险清单

**指标定义风险**：
- [ ] "延误"定义 → > 15min vs > 30min？
- [ ] 取消 vs 改签 → 取消才算？
- [ ] Disruption Index 计算方式 → 需了解公式

**非天气因素排除**：
- [ ] 航空公司机械故障 → 非天气原因
- [ ] 空中管制 → NAS Delay
- [ ] 节假日效应 → 感恩节/新年
- [ ] 军事活动 → 禁飞区

**数据源风险**：
- [ ] 各平台数据不一致 → 需多源验证
- [ ] 数据延迟 → 实时 vs T+1
- [ ] 覆盖率 → 小机场可能无数据

**因果关系风险**：
- [ ] 天气是主因吗？→ 需排除其他因素
- [ ] 预警 vs 实际 → 航班取消可能过度
- [ ] 区域 vs 局部 → 多机场影响

- [ ] 多机构数据不一致 → CMA/JMA/JTWC
- [ ] 快速增强 → 预报难度大
- [ ] 路径突变 → 打转、转向

## 结算验证检查清单

每次结算前，按以下清单逐项核对：

### 通用检查

- [ ] **时间窗**：结算时刻/时段是否明确？
- [ ] **位置**：站点/区域是否清晰？有代码？
- [ ] **阈值**：数值/单位/边界处理是否明确？
- [ ] **数据源**：官方来源是否指定？是否可获取？
- [ ] **争议**：潜在冲突点是否已识别？

### 天气特定检查

- [ ] **时区**：UTC vs 本地时？夏令时？
- [ ] **站点**：数据源与市场描述一致？
- [ ] **指标**：观测 vs 预报？瞬时 vs 时段？
- [ ] **单位**：换算是否正确？

### 航空特定检查

- [ ] **非天气因素**：排除管制/节假日/航空公司的影响？
- [ ] **指标定义**：延误/取消/Disruption Index 公式？
- [ ] **机场代码**：ICAO vs IATA？
- [ ] **数据覆盖**：目标机场是否有数据？
